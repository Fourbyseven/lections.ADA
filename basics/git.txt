git init  - Создает git репозиторий из текущей папки (пишется 1-раз в самом начале)
git add . (Либо вместо точки любой файл из директории) - добавляет файлы и папки для отслеживания в index
(Эту комманду пишем каждый раз перед коммандоай commit)
git commit -m  'сообщение'  - Создает новую версию если были изменение в отслеживающихся файлах

Полный путь от создания локального репозитория, до отправки на удаленный
1) Создаем новый репозиторий на GitHub
2) git init - Инициализируем локальный репозиторий, после этой комманды в Директорий должен появиться файл-
.git, для его просмотра можно выполнить ls -a

3) git add . - добавляет все файлы в папке в index git 
4) git commit - m 'comment' - коммитим все изменение (сохраняем их)
5) Устанавливаем связь через SSH-ключ
    1) Генерируем SSH - ключ через комманду ssh-keygen
    при созданни ключа, в сообщении будет ключ указан путь по которому ключ сохранен
    Обычно он выглядит так /home/<имя вышего пользователя>/.ssh
    2) заходим в папку .ssh и копируем содержимое файла  id_rsa.pub (cat id_rsa.pub)
    3) Заходим на GitHub Переходим в settings/настройки
    4) Заходим в пункт SSH and Gps keys, и Нажимаем кнопку New SSH key
    5) Пишем название для ключа (Может быть любым) и ниже в поле для ключа
    Вставляем 
6) git remote add origin <SHH

7) git push origin main/master - комманда которая отправляет все изменения на удаленный репозиторий на GitHub
